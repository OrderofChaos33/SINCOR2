# Uniswap funding brief

## Best-fit paths

1. Uniswap Foundation grant
2. Uniswap Hook Incubator

## Core angle

SINCOR2 already documents and ships Uniswap v4-related work, including a public `SincLimitOrderHook` and a broader Base-native liquidity architecture around SINC and AXIOM. The strongest case is not generic AI; it is reusable DeFi infrastructure plus visible hook execution in a live ecosystem context.

## Assets to foreground

- Public hook documentation and contract references
- Base-native deployment context
- Agent marketplace use case for onchain settlement and liquidity
- Treasury-routing and token-mechanics documentation

## What to prove before submission

- Why the hook or tooling is useful beyond this project
- What developers, traders, or integrators gain from the work
- Any measurable usage, test coverage, audits, or third-party verification
- Whether the grant ask is for infrastructure hardening, tooling, documentation, or ecosystem expansion

## Submission posture

- Use Foundation grant if asking for direct support to extend or harden public infrastructure.
- Use Hook Incubator if the main goal is mentorship, design iteration, and visibility for hook-specific work.
- Keep the story concrete: shipped contracts, public code, Base deployment context, and developer utility.
