1. Data Encryption

1.1 Encryption at Rest

All customer data stored on SINCOR's infrastructure is encrypted using AES-256, the industry standard for enterprise data protection. This includes:

  • Database records containing customer and business data
  • File storage for deliverables, reports, and uploaded assets
  • Backup copies and disaster recovery snapshots
  • Secrets management (API keys, credentials) using vault-based encryption

1.2 Encryption in Transit

All data transmitted between your browser/application and SINCOR's servers is protected by TLS 1.2 or higher. We enforce HTTPS on all endpoints and reject older, insecure protocols.

2. Access Controls

2.1 Role-Based Access Control (RBAC)

Access to customer data within SINCOR is governed by strict role-based permissions. Employees and systems access only the minimum data required to perform their function (principle of least privilege). All access is logged and auditable.

2.2 Customer Access Controls

Customers can manage access through:

  • Secure password authentication with bcrypt hashing
  • Session management with automatic timeout
  • Activity logs showing recent account access
  • Team-level access permissions (Enterprise plans)

2.3 Internal Access Governance

SINCOR employees undergo background checks before receiving data access. Production data access requires multi-factor authentication, is limited to authorized personnel, and is logged in immutable audit trails reviewed quarterly.

3. Infrastructure Security

3.1 Cloud Infrastructure

SINCOR runs on enterprise cloud infrastructure with:

  • Isolated, containerized deployment for each customer environment
  • Network segmentation and private VPCs
  • Web Application Firewall (WAF) protection against OWASP Top 10
  • DDoS mitigation and rate limiting
  • Automated vulnerability scanning and patch management

3.2 Physical Security

Our infrastructure partners maintain SOC 2-certified data centers with 24/7 physical security, biometric access controls, and environmental monitoring.

4. Application Security

4.1 Secure Development Lifecycle

  • All code changes undergo peer code review before deployment
  • Automated static analysis and dependency vulnerability scanning (SAST/SCA)
  • Regular third-party penetration testing
  • Secure coding training for all engineering staff

4.2 API Security

  • All API endpoints require authentication
  • Rate limiting to prevent abuse and brute-force attacks
  • Input validation and parameterized queries to prevent injection attacks
  • CORS policies restricting cross-origin access

4.3 Secrets Management

API keys, credentials, and configuration secrets are stored in a dedicated secrets management system (never hardcoded in source code). Keys are rotated regularly and revoked immediately upon employee offboarding.

5. Data Handling & PII

5.1 Data Minimization

We collect only the personal information necessary to provide the Services. Business data you provide (customer lists, content, etc.) is processed solely for the purpose of delivering your AI services and is not shared or used for any other purpose.

5.2 Data Flow

Customer data flow within the SINCOR platform:

  • Input: Your data enters via encrypted HTTPS connections
  • Processing: AI agents process data in isolated, containerized environments with no cross-customer access
  • Storage: Results stored in encrypted, customer-isolated storage
  • Delivery: Deliverables accessible only through authenticated sessions
  • Deletion: Data purged per retention schedules or upon your request

5.3 Third-Party Sub-Processors

SINCOR uses a limited set of vetted sub-processors for specific functions (cloud hosting, payment processing, email delivery). All sub-processors are bound by data processing agreements consistent with our privacy obligations. A complete list of sub-processors is available upon request by emailing privacy@getsincor.com.

6. Compliance Frameworks

GDPR (EU/UK)
Lawful basis for processing, data subject rights, 72-hour breach notification, DPA with sub-processors, Standard Contractual Clauses for international transfers
CCPA/CPRA (California)
Right to know, delete, correct, and opt out. No sale of personal information. Annual data inventory maintained.
CAN-SPAM / CASL
AI outreach agents enforce 12-touch maximum, mandatory unsubscribe handling, business hours only, and compliant sender identification.
SOC 2 Type II (In Progress)
We are working toward SOC 2 Type II certification. Our infrastructure partners maintain SOC 2-certified facilities. Expected completion: Q4 2025.

7. AI Agent Compliance & Ethics

SINCOR's AI agents operate under strict responsible-use policies:

  • Outreach limits: Maximum 12-touch sequences; 48-hour minimum intervals between touches; auto-stop on unsubscribe
  • Lead discovery: 2,000 contacts/day maximum; excludes protected lists (do-not-contact registries); respects robots.txt
  • Human oversight: High-risk actions (executive-level outreach, custom pricing, unusual transactions) require human approval
  • Data sourcing: Only public data sources used for lead discovery; no proprietary or confidential data accessed without permission
  • Transparency: AI-generated content is identifiable in your dashboard with full audit trails

8. Incident Response

In the event of a security incident:

  • Detection: 24/7 automated monitoring with real-time alerting
  • Containment: Automated isolation of affected systems within minutes
  • Notification: Affected customers notified within 72 hours of confirmed breach (per GDPR requirements)
  • Remediation: Root cause analysis and corrective actions documented
  • Post-incident review: Process improvements implemented within 30 days

To report a security vulnerability, email security@getsincor.com. We operate a responsible disclosure program and commit to responding within 48 hours.

9. Business Continuity & Uptime

  • Target SLA: 99.5% monthly uptime for platform services
  • Backups: Automated daily backups with 30-day retention; point-in-time recovery available
  • Disaster recovery: RTO (Recovery Time Objective) of 4 hours; RPO (Recovery Point Objective) of 24 hours
  • Maintenance: Scheduled maintenance during off-peak hours with advance notice

10. Security Certifications & Assessments
